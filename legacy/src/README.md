# Estrutura do Projeto

Este diretório contém o código fonte principal do sistema de advocacia, organizado seguindo as convenções PSR-4.

## Estrutura de Diretórios

- `Database/` - Classes para conexão e operações com banco de dados
- `Models/` - Modelos de dados (Clientes, Processos, Advogados, etc.)
- `Controllers/` - Controladores para gerenciar a lógica de negócio
- `Services/` - Serviços para operações complexas
- `Utils/` - Utilitários e helpers
- `Config/` - Configurações do sistema

## Convenções

- Use namespaces seguindo PSR-4
- Nomes de classes em PascalCase
- Nomes de métodos em camelCase
- Arquivos devem ter o mesmo nome da classe
